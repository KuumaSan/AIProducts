# /read <book_path> [--units N]

精读一本书（PDF），生成结构性笔记和记忆框架。

## 触发方式
- `/read books/xxx.pdf`
- `帮我读这本书`（用户发送 PDF 文件时）
- `精读一下这个`

## 参数
- `book_path`（必填）：PDF 文件路径，或用户直接发送的 PDF
- `--units`（可选）：分几个学习单元，默认根据页数自动决定（每单元约 30-50 页）

## 执行流程

### 1. 准备阶段
- 在 `books/` 下创建以书名命名的子目录
- 计算总页数，确定学习单元数
- 创建 `{book}_study_state.md`（进度追踪文件）
- 创建 `{book}_study.md`（滚动笔记文件）

### 2. 分块处理
- 大 PDF 按每块 15-30 页拆分（避免单次 token 超限）
- 拆分命令：`python3 -c "..."` 或 `qpdf --pages`
- 每块用 `pdf` 工具读取内容

### 3. 逐单元精读
每个学习单元：
1. 读取该单元对应的页面内容
2. 提取核心论点、框架、关键数据
3. 写入滚动笔记（追加到 `{book}_study.md`）
4. 更新进度文件 `{book}_study_state.md`
5. 向用户发送该单元的精读摘要

### 4. 完成阶段
- 在笔记末尾生成全书结构性决策框架
- 将框架精炼存入 `memory/investment_frameworks/` 并更新 `_index.md`
- Git commit + push

## 笔记格式
- 每单元标题 + 核心要点（bullet points）
- 重要原文引用保留
- 每单元末尾写"对后续分析的约束/影响"
- 全书完成后写"结构性记忆"总结

## 注意事项
- 精读不是翻译，要提炼判断和框架
- 保持批判性思维，标注作者观点的局限性
- 如果 PDF 是扫描件，先用 OCR 提取文本存入 `books/ocr_text/`
- 每单元结束后主动汇报进度
